\title{GPU-Sentry: Instruction-Level Semantic Detection of GPU Cryptomining Workloads}

\author{Pin-Yi Kuo}
\affiliation{%
  \institution{National Tsing Hua University}
  \city{Hsinchu}
  \country{Taiwan}
}
\email{yikuo@lsalab.cs.nthu.edu.tw}

\author{Jerry Chou}
\affiliation{%
  \institution{National Tsing Hua University}
  \city{Hsinchu}
  \country{Taiwan}
}
\email{jchou@lsalab.cs.nthu.edu.tw}

\begin{abstract}
Unauthorized GPU cryptomining consumes expensive accelerator resources and can interfere with legitimate HPC and AI workloads. Existing GPU cryptojacking defenses primarily rely on indirect evidence such as network traffic, host behavior, or GPU telemetry. These signals can reveal abnormal activity, but they do not directly inspect the GPU kernels submitted for execution.

We present GPU-Sentry, a system for instruction-level semantic detection of GPU cryptomining workloads. GPU-Sentry transparently captures GPU code objects and kernel-launch events through CUDA driver interception, reconstructs and normalizes the submitted GPU assembly, and builds stream-level execution windows over the kernel launch sequence. A sequence classifier then detects mining-like computation from normalized instruction traces. The capture layer is deployed system-wide by the administrator and monitors native and containerized CUDA workloads without requiring user-side application changes.

GPU-Sentry is designed around a synthetic-to-real evaluation setting: the detector is trained on synthetic mining and benign kernels and evaluated on real-world GPU captures. We show that normalized GPU instruction traces carry evidence of cryptomining that transfers from synthetic training to real-world captures, and that this evidence stays visible under launch-rate throttling and mixed workloads where a behavior-based GPU-telemetry baseline degrades.
\end{abstract}
